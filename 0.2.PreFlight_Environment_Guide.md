## 1. Install a terminal-friendly setup

Everything in this workshop is driven by typing plain-English prompts into **Claude Code**, which runs in a terminal. First we make sure the terminal and its package manager exist.

### macOS

1. Open the **Terminal** app (press `Cmd + Space`, type `Terminal`, hit Enter).
2. Install **Homebrew** (the macOS package manager). Paste this and press Enter:
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
3. Follow the prompts (it may ask for your Mac password — typing shows nothing, that's normal). When it finishes, close and reopen Terminal.
4. Verify:
   ```bash
   brew --version
   ```
   You should see a version number.

### Windows

Windows users have two paths. **WSL2 (recommended)** gives you a real Linux environment that matches the macOS instructions; **PowerShell** works too if you prefer.

**Option A — WSL2 (recommended):**
1. Open **PowerShell as Administrator** (click Start, type `PowerShell`, right-click → *Run as administrator*).
2. Install WSL2 with Ubuntu:
   ```powershell
   wsl --install
   ```
3. **Restart your computer** when prompted.
4. After restart, an **Ubuntu** window opens and asks you to create a username and password — do it (again, typing the password shows nothing). From now on, run the macOS/Linux commands inside this **Ubuntu** window.

**Option B — PowerShell only:**
1. Open **PowerShell as Administrator**.
2. Install **winget** package sources if prompted; winget ships with Windows 11 and recent Windows 10.
3. You'll use `winget install ...` commands where this guide gives them.

## 2. Install Node.js 18+ , Git, and VS Code

These are the three baseline tools every build uses.

### macOS

```bash
brew install node git
brew install --cask visual-studio-code
```

### Windows (WSL2 — Ubuntu window)

```bash
sudo apt update
sudo apt install -y nodejs npm git
```
Then install **VS Code for Windows** from https://code.visualstudio.com/ and, when it opens, install the **WSL** extension (it will prompt you).

### Windows (PowerShell only)

```powershell
winget install OpenJS.NodeJS.LTS
winget install Git.Git
winget install Microsoft.VisualStudioCode
```
Close and reopen PowerShell after installing.

### Verify (all platforms)

Run each line; each should print a version number:

```bash
node --version    # must be v18.0.0 or higher
git --version
code --version
```

> **If `node --version` shows something below v18** (or "command not found"): on macOS run `brew upgrade node`; on WSL/Ubuntu install the current LTS with
> ```bash
> curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt install -y nodejs
> ```

## 3. Install & log in to Claude Code

Claude Code is the primary tool for the entire day — every concept is taught and built in it.

### Install (all platforms)

```bash
npm install -g @anthropic-ai/claude-code
```

> If you get a **permissions error** on macOS/Linux, do **not** use `sudo`. Instead set npm's global folder to your home directory:
> ```bash
> mkdir -p ~/.npm-global && npm config set prefix ~/.npm-global
> echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc   # macOS
> echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc  # WSL/Ubuntu
> ```
> Close and reopen the terminal, then re-run the install command.

### Log in

1. Start it:
   ```bash
   claude
   ```
2. The first run opens your browser to sign in. **Use the Claude account (with the Pro subscription) provided for the workshop.**
3. Approve the login in the browser, return to the terminal — you should see the Claude Code prompt.

### Verify

```bash
claude --version
```
A version number = you're logged in and ready.

## 6. The 30-second smoke test (do this last)

This confirms the whole chain works end-to-end. On workshop morning you'll repeat a version of this.

1. Make and enter a scratch folder:
   ```bash
   mkdir ~/100x-smoketest && cd ~/100x-smoketest
   ```
   *(Windows PowerShell-only users: use `mkdir $HOME\100x-smoketest; cd $HOME\100x-smoketest`.)*
2. Start Claude Code:
   ```bash
   claude
   ```
3. At the prompt, type this in plain English and press Enter:
   ```
   Create a file called hello.txt containing today's date and a one-line hello message.
   ```
4. Approve the action when Claude asks.
5. Confirm the file exists:
   ```bash
   cat hello.txt          # macOS / WSL / Linux
   type hello.txt         # Windows PowerShell
   ```

**✅ You see your hello message → you are 100% ready for the workshop.**

## 7. Troubleshooting quick table

| Symptom | Fix |
|---|---|
| `command not found: claude` | The npm global bin isn't on your PATH — see the permissions note in §3, reopen the terminal. |
| `command not found: brew` (macOS) | Homebrew didn't finish; re-run the §1 install line and follow the "Next steps" it prints. |
| `wsl --install` fails (Windows) | Enable virtualization in BIOS, and turn on *Windows Subsystem for Linux* + *Virtual Machine Platform* under **Turn Windows features on or off**, then reboot. |
| Node version below 18 | See the upgrade note at the end of §2. |
| Login browser page won't open | Copy the URL Claude prints in the terminal and paste it into your browser manually. |
| Corporate laptop blocks installs | You need **admin rights**. If IT-locked, bring a personal laptop or arrive 15 min early for a TA. |
| Behind a corporate proxy / VPN | AI and dev domains (claude.ai, github.com, npm, replit.com, v0.dev) must be reachable; disable VPN for the workshop if it blocks them. |
